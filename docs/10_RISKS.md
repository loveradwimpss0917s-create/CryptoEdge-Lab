# 10. リスク一覧と対策

| ID | リスク | 影響 | 発生確度 | 対策 (設計への織込み) |
|---|---|---|---|---|
| **研究の妥当性** |
| R-S1 | 過学習・多重検定による偽 Edge 採用 | 資金損失につながる誤確信 | 高 | EEP の DSR/FDR/Permutation (docs/05 §3.5–3.7)、試行レジストリ、PAPER 30 日ゲート |
| R-S2 | look-ahead / survivorship バイアス | 再現不能な好成績 | 高 | 二時制モデル、pit_lag、次バー執行、因果レジームのみ使用、DQ-08 |
| R-S3 | Edge の公表後劣化 (PDF 由来は特に) | 期待値の過大評価 | 高 | 直近 2y セグメント必須、CUSUM 劣化監視、REJECT 条件「直近 2y EV<0」 |
| R-S4 | レジーム変化で全 Edge 同時劣化 | ポートフォリオ幻想 | 中 | レジーム別評価、相関管理、有効独立数 KPI |
| R-S5 | コスト・スリッページの過小評価 | 紙上の利益 | 中 | コスト後のみを正値表示、paper の実測スリッページ照合 (PAPER→ACTIVE ガード) |
| **データ** |
| R-D1 | 無料 API の仕様変更・廃止・地域制限 (Binance 等) | 収集停止 | 高 | アダプタ抽象化 + フォールバック宣言、DQ-02 検知、data.binance.vision 代替、複数取引所冗長 |
| R-D2 | 非公式 API (Yahoo/Trends) の ToS・ブロック | 一部系列停止 | 中 | 冗長系 (Stooq/SerpAPI) 差替え設計、重要 Edge を非公式ソースに依存させない (CME ギャップは判定要素を再確認の上、必要なら CoinGlass へ) |
| R-D3 | 清算データの不完全性 (Binance サンプリング) | EC-006 系の検証歪み | 中 | source_id 分離、CoinGlass 併用、counter_evidence 明記 |
| R-D4 | 改訂データ (ETF/M2) の遡及変更 | PIT 破れ | 中 | 二時制 + revisable フラグ + DQ-09 |
| **プラットフォーム (無料枠運用)** |
| R-P1 | **無料枠の上限変更・無料プラン廃止** (Cloudflare/GitHub の規約改定) | 収集停止・移行コスト | 中 (数年スパンでは十分起こる) | docs/13 を上限値の単一管理点にする / quota_usage で消費実績を常時可視化 / データは標準形式 (SQLite・Parquet) で保存しロックインを回避 / 有料化しても $5–10/月で吸収できる構成 (docs/13 §6) |
| R-P2 | 無料枠の予算超過 (D1 5GB・書込 100K/日・KV 1K/日・サブリクエスト 50・Actions 2,000 分) | 静かな機能停止 | 中 | **ヘッドルーム監視を本番機能化** (60%/80% 警告 + 自動緩和, docs/13 §7)。設計時点で全予算 ≤ 25% (docs/13 §1) |
| R-P3 | Cron tick の欠落・失敗 (Queues なし構成) | データ穴 | 中 | watermark 方式 + D1 `ingest_tasks` リトライ + DQ-01 リフィル。GitHub schedule は信頼せず Worker dispatch を正とする |
| R-P4 | シークレット漏洩 (API キー, PAT) | 不正利用 | 低 | Worker Secrets のみ、リポジトリに置かない、最小スコープ、Access 前段 |
| R-P5 | Workers CPU 10ms 超過 (実装ミスで重い処理を書く) | tick 失敗の頻発 | 中 | 境界規約 (docs/01 §3) + レビュー観点化 + gzip/集計は Actions へ。CPU 例外は dq_issues に記録され検知可能 |
| R-P6 | AI ハンドオフの幻覚・貼り戻し汚染 | 誤情報が研究記録に混入 | 低–中 | Pack にガードレール文 + 返信スキーマ強制 + zod 検証 + draft レビュー制 (docs/07 §2)。AI 出力は判定に使わない原則で影響半径を限定 |
| R-P7 | 非公式クライアント経路 (DuckDB-WASM の R2 Range 読み) のブラウザ互換問題 | Explorer 不能 | 低 | 定型ビューは全て API 側にあり Explorer なしでも研究は成立 (探索の利便性のみ低下) |
| **プロジェクト** |
| R-J1 | EEP 実装バグ (最悪の失敗モード: 静かに間違う) | 全判定が無効 | 中 | ゴールデンテスト・既知分布シミュレーションテスト (docs/11 §3)、protocol_version で再評価可能 |
| R-J2 | スコープ肥大 (54 件全部を最初に検証したくなる) | V1 遅延 | 高 | P0 5 件固定、残りは IDEA 登録のみ (docs/09) |
| R-J3 | 法規制 (投資助言該当性) | — | 低 | 単一ユーザ・自己研究用途。共有機能を作らない (V3 マルチユーザ化時に再評価) |

**受容するリスク**: 無料データの品質限界 (V1)、1 分未満の現象は対象外 (V1–V2)、Cloudflare ベンダーロックイン (R2/D1 は標準形式でエクスポート可能なため実害小)、ブリーフィングの文章品質がテンプレ調であること (AI ハンドオフで必要時のみ補完 — 固定費ゼロの対価として受容)。
